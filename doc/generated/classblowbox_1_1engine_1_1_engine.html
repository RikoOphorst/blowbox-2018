Detailed Description

The class that initializes & manages all systems in the engine and also controls the game loop.

If you want to boot the blowbox engine, you should create an instance of this class. To run the engine, you should call Run(). To shutdown the engine, you should call Shutdown(). If you want to add custom logic to certain events that happen throughout the lifetime of the engine, you can set callback functions using any of the SetOn* functions. If you want to supply any custom data to any of these callback functions, you can set a custom void* to some sort of userdata object you would like to pass along to all of your callbacks.

Remarks
Only one instance of the Engine can be alive at the same time.
You can easily retrieve the current instance of the engine via Engine::Instance().

void blowbox::engine::Engine::Run ( )

Initializes all the systems in the engine and starts running the gameloop.

Remarks
This function is blocking.
You can set callbacks for specific events that happen during Engine runtime by using the various Engine::SetOn* functions.
You can set a custom userdata pointer by calling Engine::SetCalllbackUserdata() which gets sent to every callback.
You should set all your callbacks before calling Engine::Run(), although it is possible to modify them later on during engine runtime.
